# Macros ETL Project

Проект для автоматизации загрузки, обработки и выгрузки отчетных данных из Excel-файлов в ClickHouse.

Сейчас проект решает две основные задачи:

1. **Weekly pipeline**  
   Берет данные из источников **MO** и **MX**, обрабатывает их, формирует итоговый датасет, выгружает результат в Excel и загружает подготовленные данные в ClickHouse.

2. **Raw daily pipeline**  
   Каждый день читает “сырые” Excel-файлы из актуальных папок MO и MX, приводит данные к нужному формату и сохраняет их в raw-таблицы ClickHouse без бизнес-агрегации.

Дополнительно в проекте есть отдельный разовый backfill-скрипт для исторической загрузки `raw_mo_daily` из архивных папок.

---

## Бизнес-назначение проекта

Один из ключевых результатов weekly pipeline — Excel-файл для поиска **нарушений мест отбора**.

Под нарушением места отбора понимается ситуация, когда сборщики забирают товар из мест хранения с **большим сроком годности**, хотя в местах отбора уже есть товар с **меньшим остаточным сроком годности**.

Иными словами, проект помогает выявлять случаи, когда товар отбирается не по правильной логике ротации сроков годности, что может приводить к:
- накоплению более “старого” товара в местах отбора,
- росту риска списаний,
- нарушению FIFO/FEFO-подхода,
- ухудшению качества операционного контроля.

Итоговый Excel-отчет используется как инструмент контроля и анализа таких отклонений.
